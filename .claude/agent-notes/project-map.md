# Карта проекта (v10.1 — UI-чистка после v10)

> **v10.1-дельта (UI-чистка по скрину):** серия мелких правок поверх v10. Логика данных не тронута. (1) Из `EquityChart.tsx` удалена интерактивная легенда с именами/балансами участников + кнопка «сбросить» (дублировали боковую панель и end-points), вместе с пропом `onFocusChange` и локальной `setHighlight` — выделение полностью контролируется родителем через URL `?focus=`. Маленькая легенда «закрытые сделки» оставлена. (2) Высота canvas графика: `h-[420px] sm:h-[520px] lg:h-[580px]` (было `340/420`). (3) Из тулбара `DashboardShell.tsx` убран UI-индикатор «● live выкл / обновлено N сек назад»; стейл-сигнал переехал в `console.warn('[useLivePrices] stale: …')` (только когда `live.stale && live.error`). (4) Мета-подпись под графиком теперь живая: `📊 ${note}` + ` · до следующего обновления: M:SS` (тикает 1 раз/сек, источник = `live.fetchedAt + NEXT_PUBLIC_PRICES_REFRESH_MS - now()`); если поллить нечего — суффикс пуст; если канал стейл — суффикс ` · обновление вручную`. (5) Из `app/page.tsx` удалён правый ярлык шапки «Реалити-торговля · live-дашборд» (был дублем `competition.title`). (6) Удалён `<footer>` с `competition.title · YYYY` (тот же дубль). Контракты внешних компонентов не менялись (только `EquityChart` потерял проп `onFocusChange`, и DashboardShell перестал его передавать).

> **v10-дельта (live-sync + focus в URL):** новый клиентский хук `app/components/useLiveCompetition.ts` подписывается через Supabase Realtime на `postgres_changes` по 5 таблицам (`participants`, `positions`, `balance_points`, `competition_meta`, `tickers`) и при любом изменении вызывает `router.refresh()` (с дебаунсом 400 мс) → RSC `getCompetitionData()` перезапрашивается, новый payload мёрджится в DashboardShell без потери client state. Если Realtime канал не открылся (CHANNEL_ERROR / TIMED_OUT) или env Supabase не задан — graceful degradation: при наличии env → polling-fallback `setInterval(router.refresh, 45_000)`; при отсутствии env (статика) → no-op. Хук подключён ОДНОЙ строчкой в `DashboardShell.tsx`. Новая миграция `supabase/migrations/0007_realtime.sql` — идемпотентный `alter publication supabase_realtime add table ...`. Никаких новых production-deps (Realtime внутри `@supabase/supabase-js`). Также: верхний дропдаун-фильтр участников УБРАН, заменён на «focus»-фильтр (выделение одного участника на графике + dim остальных). Источник правды focus — URL `?focus=<имя>` через `useSearchParams`/`router.replace`. Стартовое значение читается на сервере в `app/page.tsx` (новый проп `searchParams: Promise<...>`) и пробрасывается `initialFocusedName` в `DashboardShell` — гидрационная консистентность. Закрытие `ParticipantModal` теперь автоматически устанавливает focus на этого участника (по спеке). `EquityChart` получил пропсы `focusedName` + `onFocusChange` (раньше держал локальный `highlight` сам — теперь полностью контролируется родителем). Карточка в боковой панели (`ParticipantCard`) получила пропсы `focused` (обводка цветом + градиент + стрелка `◀`) и `dimmed` (`opacity-40`). В заголовке графика — подсказка «график: <имя>» / «график: все» + кнопка «Все» для сброса. `app/page.tsx` теперь `async`, читает `searchParams` (Promise в Next 16).

> **v9.1-дельта (admin entry_price + demo seed):** в форму `/admin` (open + edit trade) добавлено опц. поле `entry_price` (numeric, шаг подбирается по инструменту: 0.0001 / 0.01 / 1). Подсказка под формой объясняет: задано → live-PnL из TwelveData; пусто → ручной unrealized_pnl. `actions.ts` (`openTrade` / `updateTrade` / `getAdminData`) умеет работать со старой схемой (без колонки) через try/catch + fallback. `data/competition.ts` перезалит на 6 русских участников (Андрей/Михаил/Алексей/Дмитрий/Сергей/Иван), депозиты 1200–5000 $, у открытых позиций — entry_price ≈ актуальной рыночной цене на 2026-05-13 (XAU 4640 / EUR 1.0850 / GBP 1.2700 / USDCAD 1.3850 / USDJPY 152 / BTC 80000). startDate сдвинут на 2026-05-06. `supabase/migrations/0005_demo_data.sql` — те же данные для Supabase (прогнать вручную).

Тулбар: индикатор «обновлено N сек назад / данные на HH:MM / live выкл». ## Где что менять

- **Подключить Supabase** → заполнить `.env.local` из `.env.example` + прогнать `supabase/migrations/0001_init.sql` и `0002_watchlist.sql`
- **Изменить схему БД** → новый файл `supabase/migrations/000N_*.sql` + обновить типы в `lib/types.ts` и маппинг в `lib/db.ts`
- **Без БД — данные** → `data/competition.ts` (timeline / tickers / participants / watchlist / title / startDate / endDate / note)
- **Логика загрузки данных / fallback** → `lib/db.ts`
- **Вид графика** → `app/components/EquityChart.tsx`
- **Раскладка дашборда, табы, focus-фильтр (URL ?focus=), карточки участников** → `app/components/DashboardShell.tsx`
- **Live-sync (Supabase Realtime → router.refresh)** → `app/components/useLiveCompetition.ts` + миграция `supabase/migrations/0007_realtime.sql`
- **Шапка / порядок блоков** → `app/page.tsx`
- **Строка тикеров (UI + опрос)** → `app/components/TickerStrip.tsx`; **источники live-цен** → `app/api/tickers/route.ts`
- **Лидерборд** → `app/components/Leaderboard.tsx` + `getLeaderboard()` в `lib/standings.ts`
- **Расчёты статистики, форматтеры** → `lib/standings.ts`
- **Формы админки** → `app/admin/AdminPanel.tsx` (UI) + `app/admin/actions.ts` (логика/мутации)
- **Цвета темы** → `app/globals.css`

## Конвенции
- alias `@/` → корень проекта (`tsconfig.json`)
- Всё UI — по-русски, кроме тикеров (XAUUSD и т.п.) и LONG/SHORT
- recharts и любой client-state → "use client" компонент (EquityChart, DashboardShell, AdminPanel)
- `lib/supabase.ts` / `lib/db.ts` / `app/admin/actions.ts` — ТОЛЬКО server (никогда не импортировать из "use client"); service-role ключ не утекает на клиент
- Server Actions для мутаций (нативный FormData), без библиотек форм
